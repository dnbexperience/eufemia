## Component changes

### TODO: Remove unused CSS custom properties

> **This task has not been completed yet.** The properties listed below are still present in the codebase and documentation.

The following CSS custom properties are defined but never consumed via `var()` in the codebase. They are currently documented and should be removed along with their documentation:

**Layout breakpoints** (documented in [media queries](/uilib/layout/media-queries) and v10-info.mdx):

- `--layout-small` (`40em`)
- `--layout-x-large` (`80em`)
- `--layout-xx-large` (`90em`)

**Shadow partials** (documented in [helper classes](/uilib/helpers/classes)):

- `--shadow-default-blur-radius` (`16px`)
- `--shadow-sharp-x` (`0`)
- `--shadow-sharp-y` (`1px`)
- `--shadow-sharp-blur-radius` (`6px`)
- `--shadow-sharp-color` (`rgb(0 0 0 / 16%)`)

**Typography** (documented in [styling](/uilib/usage/customisation/styling)):

- `--font-weight-default` (`400`)

**Colors** (documented in color table):

- `--color-mint-green-25` (`#e0f5f0`)

When removing these properties, also update the documentation in `classes.mdx` (shadow helper classes), `media-queries.mdx`, `styling.mdx`, and `ColorTable.tsx` accordingly.

### [Accordion](/uilib/components/accordion/)

- Replace `id` with `connectedTo` on `Accordion.Content` when connecting to a standalone tertiary button.

### statusState â†’ intent

The `statusState` prop has been deprecated in favor of `intent`. Replace `statusState` with `intent` on all form components that support it.

```diff
-<Input status="Error message" statusState="error" />
+<Input status="Error message" intent="error" />
```

The `intent` prop also supports setting visual intent without a status message. When used this way, consider adding `aria-label` or `aria-labelledby` to convey the intent to assistive technologies:

```jsx
<Input intent="error" aria-label="Error in field" />
```

Affected components: Autocomplete, Button, Checkbox, DatePicker, Dropdown, Input, Radio, RadioGroup, SegmentedField, Slider, StepIndicator, Switch, Textarea, ToggleButton, ToggleButtonGroup.

### [Modal](/uilib/components/modal/), [Dialog](/uilib/components/dialog/) and [Drawer](/uilib/components/drawer/)

- Replace `closeButtonAttributes` with `closeButtonProps`.

```diff
-<Drawer closeButtonAttributes={{ text: 'Custom text' }} />
+<Drawer closeButtonProps={{ text: 'Custom text' }} />
```

- Replace `triggerAttributes` with `triggerProps`.

```diff
-<Dialog triggerAttributes={{ text: 'Open', variant: 'primary' }} />
+<Dialog triggerProps={{ text: 'Open', variant: 'primary' }} />
```

- The `ModalTriggerAttributes` type has been removed. Use `ButtonProps` from `@dnb/eufemia` instead.

### [InfoCard](/uilib/components/info-card/)

- Replace `closeButtonAttributes` with `closeButtonProps`.

```diff
-<InfoCard closeButtonAttributes={{ href: '/path' }} />
+<InfoCard closeButtonProps={{ href: '/path' }} />
```

- Replace `acceptButtonAttributes` with `acceptButtonProps`.

```diff
-<InfoCard acceptButtonAttributes={{ href: '/path' }} />
+<InfoCard acceptButtonProps={{ href: '/path' }} />
```

### [Popover](/uilib/components/popover/)

- Replace `triggerAttributes` with `triggerProps`.

```diff
-<Popover triggerAttributes={{ 'aria-haspopup': 'menu' }} />
+<Popover triggerProps={{ 'aria-haspopup': 'menu' }} />
```

- The `PopoverTriggerAttributes` type has been removed. Use `triggerProps` instead.
