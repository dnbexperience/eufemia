## Compound Component Properties

### Toaster.Host

The core engine. Manages all message state, renders the toast stack, and provides screen reader announcements. Always required.

<PropertiesTable props={ToasterHostProperties} />

### Toaster.NotificationCenter

The popup panel that displays all messages. Opens when the trigger button is clicked. **You must include either this or `NotificationCenterList`.** An error is thrown at runtime if neither is present.

<PropertiesTable props={ToasterNotificationCenterProperties} />

### Toaster.NotificationCenter.Button

The button that opens and closes the notification center. Includes a hidden focus proxy that receives focus when the notification center closes and routes it back to the button.

<PropertiesTable props={ToasterNotificationCenterButtonProperties} />

### Toaster.NotificationCenterList

Renders messages inline in the document flow. Use this instead of `NotificationCenter` when you want to build your own notification center UI. **An alternative to `NotificationCenter` — you must include one or the other.**

<PropertiesTable props={ToasterNotificationCenterListProperties} />

### Toaster.FocusProxy

A hidden element that receives focus when the notification center closes and routes it back to the notification button. Only needed in compound setups where `NotificationCenter.Button` is not used.

<PropertiesTable props={ToasterFocusProxyProperties} />
