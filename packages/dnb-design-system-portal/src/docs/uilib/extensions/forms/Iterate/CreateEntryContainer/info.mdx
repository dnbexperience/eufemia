## Description

`Iterate.CreateEntryContainer` enables users to create a new entry in the array. It can be used instead of the [PushButton](/uilib/extensions/forms/Iterate/PushButton/).

It allows the user to fill in the fields without storing them in the data context.

Fields inside the container must have an `itemPath` defined.

You can place it under the [Array](/uilib/extensions/forms/Iterate/Array/) component like this:

```tsx
import { Iterate, Field } from '@dnb/eufemia/extensions/forms'

render(
  <>
    <Iterate.Array>...</Iterate.Array>

    <Iterate.CreateEntryContainer title="New item title">
      <Field.Name.Last itemPath="/name" />
    </Iterate.CreateEntryContainer>
  </>,
)
```

Technically it uses the [EditContainer](/uilib/extensions/forms/Iterate/EditContainer/) and the [Form.Isolation](/uilib/extensions/forms/Form/Isolation/) under the hood.

## Show a button to create a new entry

By default, it keeps the form open after a new entry has been created. You can change this behavior by using the `showButton` and `showButtonWhen` props.

These props allow you to render a button (`showButton`) and determine when to show it based on the logic provided by the `showButtonWhen` function. The `showButtonWhen` function receives the current list of items as an argument.

The button will be shown instead of the content provided by the children when the `showButtonWhen` function returns `true`.

```tsx
import { Iterate, Field } from '@dnb/eufemia/extensions/forms'

render(
  <>
    <Iterate.Array>...</Iterate.Array>

    <Iterate.CreateEntryContainer
      title="New item title"
      showButton={
        <Iterate.CreateEntryContainer.OpenButton text="Add another item" />
      }
      showButtonWhen={(list) => list.length > 0}
    >
      Will be hidden based on the showButtonWhen function
    </Iterate.CreateEntryContainer>
  </>,
)
```

The `Iterate.CreateEntryContainer.OpenButton` accepts the same props as the [Button](/uilib/components/button/) component.
