# Type-checked paths

Eufemia Forms uses [JSON Pointer](/uilib/extensions/forms/getting-started/#what-is-a-json-pointer) strings (such as `/firstName`) for the `path` prop. By default these are plain strings, so a typo like `/firstNme` is not caught until runtime.

With `createForm<MyData>()` you bind your data type once and get **autocomplete** and **compile-time type checking** on the `path` prop of every `Field.*` and `Value.*`, while `Form.Handler` keeps its typed `data` and `defaultData`.

## Usage

`createForm<MyData>()` returns the same `Form`, `Field` and `Value` you already know — only the `path` prop is narrowed to the valid paths of `MyData`. The rest of the `Form` namespace (`Form.Card`, `Form.SubmitButton`, …) stays available unchanged.

```tsx
import { createForm } from '@dnb/eufemia/extensions/forms'

type MyData = {
  firstName: string
  age: number
  address?: {
    street: string
  }
}

const { Form, Field, Value } = createForm<MyData>()

function MyForm() {
  return (
    <Form.Handler defaultData={{ firstName: 'Nora', age: 30 }}>
      {/* Autocompleted and type-checked */}
      <Field.String path="/firstName" />
      <Field.Number path="/age" />
      <Field.String path="/address/street" />

      {/* Type error: "/unknown" is not a path in MyData */}
      <Field.String path="/unknown" />
    </Form.Handler>
  )
}
```

## Good to know

- The typing is purely type-level and is erased at build time, so the types themselves add no runtime or bundle-size cost.
- Nested objects and arrays are supported, e.g. `/address/street` or `/items/0/title`.
- Paths are typed against the root data type. Relative paths used inside [Form.Section](/uilib/extensions/forms/Form/Section/) and [Iterate](/uilib/extensions/forms/Iterate/) are therefore not narrowed.
- Tree-shaking tradeoff: `createForm` returns the `Form`, `Field` and `Value` namespaces as runtime values, so bundlers pull in the full Forms field set (verified with both esbuild and Rollup). If bundle size is critical, keep the standard `import { Field } from '@dnb/eufemia/extensions/forms'` usage (which tree-shakes under Rollup/Vite) and apply the exported `Paths<MyData>` type only to the path values.
- See also the [TypeScript support](/uilib/extensions/forms/getting-started/#typescript-support) section in Getting started.
